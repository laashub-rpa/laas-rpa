runner独立运行在目标机器上

目标机器是指SUA程序最终作用于的设备

runner 基于守护进程制度, 使用双线保活, 开机自启

runner有三种接入方式: server、designer、local

优先级分别为: local > designer > server

对应含义:

|  操作方  |         含义         |
| :------: | :------------------: |
|  local   | 本地悬浮界面/web页面 |
| designer |    设计器指定接入    |
|  server  |    服务端指定接入    |



认证授权

接收&拉取指令

检查&执行指令

记录运行状态和操作记录到本地

同步本地数据到服务端





资源占用更加轻量级: 内存、硬盘、CPU、GPU、网络、文件操作数量、进程、线程、界面

安装应该尽可能简单, 自动安装、更新、启动
